Hi {{ current_user.server_username }}, this is how our team found it most useful to flag variants and samples:
  • Diagnostic Finding DF: this is used for variants that reach Pathogenic/Likely-Pathogenic (P/LP) ACMG classification in a sample with matching phenotype with the gene association (thus clinically relevant)
  • Secondary Finding SF: this is used for variants that reach P/LP ACMG classification in a gene that doesn't match sample's phenotype but it's part of ACMG 59 actionable genes (thus clinically relevant)
  • Negative Result NR: this is used for variants that does not reach P/LP ACMG classification that you want to discard
  • Unknown Significance US: this is generally used for variants that reach Unknown Significance (US) ACMG classification that you want to flag in order to keep track of them

You will also find the same values but in Temporary status (paler color), we normally use them when we want to come back to the variant/sample in a future moment:
  • Diagnostic Finding (temporary) DFt: when close to Diagnostic Finding (defined above), but needing further review
  • Secondary Finding (temporary) SFt: when close to Secondary Finding (defined above), but needing further review
  • Negative Result (temporary) NRt: when close to Negative Result (defined above), but needing further review
  • Unknown Significance (temporary) USt: when close to Unknown Significance (defined above), but needing further review
  • In Progress IP: when you have no idea about classification and want to come back later

You also have a way to Restore blank status (no flag) if you changed your mind: restore


This is how we imagined and used it, but if you have any better idea please share it with us, so we can evaluate togheter how to improve it! 😘
